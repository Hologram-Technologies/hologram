# hologram — Verification & Validation

> **Scope.** What hologram verifies and how. Every part of hologram is
> validated against an **external authority** (a published spec or an
> independent reference implementation) — never against hologram itself —
> and verified to **perform without bottlenecks**. This mirrors the
> discipline of `uor-addr` (realizations vs RFC/spec + cross-tool, TC-05
> replay) and `prism-btc` (SHA vs FIPS-180-4, merkle vs rust-bitcoin,
> mining vs real mainnet, cost-model vs χ² baselines, algebra vs Lean).
>
> The normative invariant catalog is [`CONFORMANCE.md`](CONFORMANCE.md).
> The full suite is reproducible via `just vv`.

## Principle: external ground truth, not self-reference

A test that checks a kernel against hologram's *own* reference evaluator,
or a hash against a constant hologram itself produced, proves only
internal consistency. V&V here means conformance to an authority we did
**not** author:

| part | external authority |
|---|---|
| σ-axis hash (`HologramHasher` = `prism::crypto::Blake3Hasher`) | the upstream **BLAKE3** reference crate (the algorithm authors' impl) |
| content addresses (κ-labels) | **uor-addr** (itself externally validated) + TC-05 replay |
| numeric kernels (matmul, conv, norm, softmax, gelu, …) | **ONNX operator spec** + ONNX backend node test data; **IEEE-754** |
| model ingest (GGUF / ONNX) | the **GGUF / ONNX format specs** + cross-tool (independent loaders) |
| quantization (I8 / I4 dequant) | the **GGML / ONNX** quantization reference |
| full-graph execution | an independent runtime's outputs (**ONNX Runtime / PyTorch**) |
| performance | measured baselines + budgets; no part is a bottleneck |

## V&V axes (reproducible: `just vv`)

1. **Architecture** — `cargo fmt --check`, `cargo clippy --all-targets -D warnings`, `cargo test --workspace`.
2. **Correctness conformance** — the `conformance` test targets that check each part against its external authority (class AS today; KC/MA forthcoming — see `CONFORMANCE.md`).
3. **Replay (TC-05)** — every minted κ-label's `AddressWitness::verify()` re-certifies via `prism::replay::certify_from_trace` (QS-05 fingerprint equivalence) without re-running the σ-axis.
4. **Performance / no-bottleneck** — `cargo bench` with per-part baselines and budgets (class PV); a part regressing past its budget fails V&V.
5. **Portability** — `just wasm` + `just embedded` (the lib stack builds on `wasm32-unknown-unknown` and `thumbv7em-none-eabi`, `no_std`).
6. **Docs** — rustdoc with intra-doc links denied.

## Status (this is a living document — gaps are tracked, not hidden)

- **AS (σ-axis / addressing): VERIFIED.** `HologramHasher` + `address_bytes` validated byte-for-byte against the BLAKE3 reference crate across chunk/subtree boundaries, plus determinism, collision-sensitivity, streaming.
- **MA / RP (model addressing + replay): VERIFIED.** GGUF model → verifiable κ-label, TC-05 witness round-trips; cross-tool agreement inherited from uor-addr's gguf/onnx realization (which it cross-validates against independent tools upstream).
- **KC / SC (kernel conformance + scaling): VERIFIED.** matmul, softmax, layer/RMS norm, gelu/silu, conv2d, attention, dequantize, reduce, max/avg-pool — each vs an **independent f64 reference of the ONNX op definition** (not hologram's own evaluator), across scale incl. non-power-of-2; content-addressed execution conforms + reuses byte-identically across 8³…128³.
- **Zero-movement substrate: VERIFIED (architecture + perf).** The runtime is one content-addressed buffer pool: a value lives in one aligned buffer, a slot binds to it, reuse/retention/constants are pointer-level (no per-node copy to/from a separate store, no copy-back on reuse). The legacy fixed-arena + content-store dual path and its per-node `to_vec()` movement are gone; dead paths (`Executor`, `async_session`) deleted. Boundary I/O (caller bytes, output bytes) and the one-time constant load are the only copies. Bounded for arbitrary run length via generational eviction (SC-3). Measured effect: served-reuse and whole-graph memo paths ~30% faster, no novel regression. **Zero-cost walk:** the per-execute scratch (slot→label, output→witnessed) is reused across runs (no per-execute heap growth), and `Workspace::split_borrow` returns a stack `SplitReads = SmallVec<[&[u8]; 4]>` (no per-kernel-dispatch heap allocation) — the hot path allocates only the value buffers themselves.
- **CA (content-addressed execution): VERIFIED.** Output labels are built by **witnessed *non-commutative* composition** (`compose.rs`'s `compose_ordered_blake3`, a prism `PrismModel` bound to `AddressResolverTuple<Blake3Hasher>` — the ordered ADR-061 operator uor-addr doesn't ship, realized as `prism-btc` demonstrates) and carry replayable TC-05 witnesses; determinism + order-sensitivity proven.
- **SG (sub-graph addressing): VERIFIED.** Every node is keyed on the cheap, order-sensitive `derive_label` of its operands' labels with its op signature (hot-path, `O(operands)`), so reuse extends past whole-graph exact-match: an unchanged sub-graph is elided across runs with a *different* top-level input set (SG-1, the prefix/KV case), and a common subexpression is computed once within a single run (SG-2). Output-port addresses remain the witnessed form (CA-3), minted once per output rather than per node. Elision is proven by per-walk dispatch/skip counters; the reused result is held to the independent f64 reference.
- **FU (content-addressed fusion): VERIFIED.** The UOR-native execution pass collapses `matmul → elementwise-activation` into one fused op (activation in the matmul epilogue), eliding the activation's intermediate from materialization *and* addressing — the composite carries a single κ-derivation. FU-1: fused result == independent f64 reference across scale, intermediate elided (one kernel, not two). FU-2: fusion is byte-identical to the unfused computation and is guarded (a matmul whose output has a second observer is not fused). FU-3: the production transformer-MLP fuses one matmul→gelu per layer with PV-4's floors intact. FU-4: the transformer **residual** fuses too — `matmul → add(out, residual)` becomes one `MatMulAdd` (residual added in the epilogue), eliding the separate bandwidth-bound add pass; equals the f64 reference, guarded (second-observer + the residual must be ready no later than the matmul's level), so an MLP layer collapses from four ops to two (the stack: 12 → 8 kernels).
- **WS (warm-start / cold-cache elimination): VERIFIED (lattice + fold + persisted store).** A κ-label is a deterministic function of the compiled graph, so every node whose transitive inputs are all constants (the *constant-only cone* — weight preprocessing, dequant, bias/transpose folds) has a determined label *and* result. The runtime **derives** the cone **lattice** itself at load (`InferenceSession::warm_lattice`), *post-fusion* so it always matches what the walk dispatches — no redundant copy is baked into the archive. WS-1 proves each lattice label equals an independent reference derivation, the lattice covers exactly the cone (complete), is deterministic, and a warm-loaded session matches the f64 reference. The **fold** (WS-2) materializes the cone through the real runtime (`hologram_exec::fold_archive` → `InferenceSession::materialize_cone`, the same kernels and pool as a normal execute) and re-emits the archive with a `WarmStart` results section; a session loaded from the warmed archive pins them under their lattice labels, so the **existing** residency check in the node walk elides the whole cone on the **first** run (`last_dispatched` drops by the cone size) — no walk changes, no second code path. Because the lattice is post-fusion, a fused constant-only `matmul→activation` warms too (verified). The CLI `compile` folds by default (`--no-warm` opts out). The **persisted κ-store** (WS-3: `WarmStore` trait — in-memory `MemWarmStore`, `std`-gated file-backed `FileWarmStore`) extends this across the process boundary: keyed by the lattice labels, a fresh session warms its cone from a prior run's results (even from a labels-only archive), with miss-safety (absent ⇒ recompute) and corruption-safety (failed integrity check ⇒ recompute, never a wrong result). Warm-start is observationally invisible (determinism ground, ADR-030; same discipline as FU-2 / CA-2) and the values are held to the independent f64 reference. The single mechanism throughout is the pool's residency check — warming pins, the walk elides; there is no second execution path.
- **WL (weight-layout monomorphism): VERIFIED.** The compiler panel-packs a matmul's *constant* f32 weight (consumed by that matmul alone) into the cache-oblivious leaf's contiguous-read layout at compile time (`layout::pack_b_panels_bytes`), marking the call `b_packed` and storing the packed bytes as the weight's representation — part of the single monomorphism the model compiles to, with **zero runtime copy**. WL-1 proves the packing fires (the compiled `MatMul` carries `b_packed`, the stored body is the packed extent) and is **semantics-preserving** (packed-weight output == independent f64 reference, incl. non-multiple-of-panel `n`); WL-0 holds the packed leaf to a naïve product across odd shapes. The packed weight is content-addressed by its bytes like any constant, so it composes with warm-start (WS) and fusion (FU). This is the data-representation half of "efficiency is structural" (see PV).
- **PV (performance / no-bottleneck): VERIFIED — microbench *and* production workload.** Release-only perf tests: matmul throughput floor + cubic envelope (PV-1), machine-independent reuse-vs-recompute ratio ≥8× (PV-2), conservative per-part floors for the other heavy kernels conv2d + attention (PV-3), and a **production-representative** stacked transformer-MLP (PV-4) that sustains a throughput floor under cold load, proves throughput does not collapse across sizes (arbitrary models), and shows serving reuse collapsing latency ≥8× — reporting GFLOP/s, FLOP/core-cycle, and ms/inference. Observed Zen3 (32-FLOP/cycle peak): the bare f32 matmul holds **~87–90% of peak through 256³** (28 FLOP/cycle) and the production MLP-stack reaches **~51% of peak at d=256** (16.3 FLOP/cycle), served reuse ≫1000×. The `production` criterion bench mirrors the workload across sizes (now incl. 256³/512³).

  **Efficiency is structural, not hand-tuned (lattice recursion + weight-layout monomorphism).** The matmul is a **cache-oblivious recursive decomposition** (`cpu::simd::matmul_f32_recursive`: halve the largest of m,n,k; register-tile leaf) — blocking for every cache tier emerges from the subdivision, so there is **no per-cache block constant** and efficiency does not collapse as the working set grows (the no-contrived-ceiling property, the kernel-level analog of prism's term/shape recursion). It is **zero-copy / zero-alloc** (pointer + stride into the existing aligned buffers). Constant weights are **panel-packed at compile time** (`pack_b_panels`, the data-representation half of the single monomorphism the model compiles to; `MatMulCall::b_packed`), so the leaf streams B contiguously with **no runtime copy** and no strided-gather misses — this is what lifts the production stack (d=256: +74% over the row-major baseline). Both are held to the f64 reference (`cpu::simd::tests::packed_b_matmul_matches_naive`, SC-4 constant-weight matmul, KC/SC across scale). The residual remaining-gap to peak on the *stack* is the bandwidth-bound residual/activation passes (roofline-limited by arithmetic intensity, not a hologram ceiling).
- **PA (parallel execution / whole-processor leverage): VERIFIED.** Parallelism is the *same* lattice recursion read as a task DAG: M/N splits of the cache-oblivious matmul are disjoint-output, independent sub-products. The in-tree `std`-only worker pool (`cpu::parallel`, `--features parallel`) cuts the recursion at ≈one tile/core (`output_tiles`), each tile running the **sequential** recursion on its **private L2** — so the cache hierarchy *compounds* across cores rather than contending. PA-1 proves it's observationally invisible (parallel == f64 reference, deterministic across runs — the data-race stress); PA-2 proves the pool genuinely runs tasks concurrently (`width` spins finish in `< 0.6×` their serial sum) — a regression guard for a `while let Some(t)=lock().pop_front()` footgun that had been holding the queue lock across `t()` and **silently serializing the pool** (4×100 ms tasks took 400 ms; fixed → 100 ms). Measured: bare 256³/512³ matmul ≈ **1.85× on 4 cores**, the production MLP-stack 46.9 → 52.9 GFLOP/s; small ops stay single-core (below `PAR_THRESHOLD`) so the single-core path stays optimal. Feature off by default — single-core unaffected.
- **Cache-hit maximization across workloads: VERIFIED (machine-independent, PV-1c).** No hardware counters needed: the cache-oblivious recursion's signature is that per-FLOP throughput does **not** collapse from L2-resident (128³) to far-past-L2 (512³). 512³ retains ~90% of 128³'s GFLOP/s (floor 60%); a cache-blind kernel cliffs. This verifies the UOR hierarchical model keeps misses compulsory (not capacity) across all workload sizes.
- **NS (portability): VERIFIED** via `just wasm` / `just embedded`.

There are no deferred V&V items. KC validates every kernel against the ONNX operator definition (the authority) via an independent f64 reference across scale; the official ONNX node-test corpus is that same authority repackaged as fixed vectors — a redundancy, not a remaining obligation.
