# hologram-archive

`.holo` archive format: rkyv zero-copy serialization, mmap loading, and execution entrypoints.

## Format

A `.holo` file has three sections:

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  Header (fixed)  в”Ӯ  Magic bytes, version, section offsets
в”ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
в”Ӯ  Graph section   в”Ӯ  Serialized HoloGraph (rkyv)
в”ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
в”Ӯ  Weight section  в”Ӯ  Quantized weight tensors (rkyv)
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

All persistent types derive `rkyv::{Archive, Serialize, Deserialize}`. There is a single format version вҖ” no backwards compatibility.

## Modules

### `format`

- `HoloHeader` вҖ” file header with magic, version, and byte offsets for each section
- `HoloGraph` (archive representation) вҖ” rkyv-serialized graph
- `mod` вҖ” format constants and version checks

### `loader`

- `MmapLoader` вҖ” maps a `.holo` file into virtual memory; zero-copy access to weights
- `BytesLoader` вҖ” loads from an in-memory `&[u8]` slice (useful for WASM)
- `LoadPipeline` вҖ” chains header validation вҶ’ graph deserialization вҶ’ weight loading
- `LoadPlan` вҖ” describes which sections to load (lazy loading support)

### `writer`

- `HoloWriter` вҖ” serializes a `HoloGraph` + weights to a `.holo` byte stream
- `PipelineWriter` вҖ” streaming writer for large models

### `section`

- `SectionTable` вҖ” index of byte ranges for each section
- `mod` вҖ” section type enum

### `weight`

- `WeightTensor` вҖ” rkyv-serialized quantized weight array
- `quantize` вҖ” converts `f32` weights to ring-encoded `u8` arrays

### `checksum`

SHA-256 integrity check on the weight section, verified at load time.

### `entrypoint`

- `ExecutionEntrypoint` вҖ” bundles a loaded graph + schedule ready for `hologram-exec`
- `schedule` вҖ” derives the execution schedule from the deserialized graph

## Usage

```rust
use hologram_archive::loader::MmapLoader;

let loader = MmapLoader::open("model.holo")?;
let entrypoint = loader.load_entrypoint()?;
// Pass entrypoint to hologram-exec
```
