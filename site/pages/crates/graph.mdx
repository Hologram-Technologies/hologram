# hologram-graph

Expression graph, subgraph decomposition, fusion passes, and parallel scheduling.

## Modules

### `graph`

- `HoloGraph` — directed acyclic graph of `Node` and `Edge` values
- `Node` — wraps a `PrimOp` with input/output shape metadata
- `Edge` — typed connection between nodes
- `validate` — structural invariant checks (no cycles, shape compatibility)

### `subgraph`

- `Subgraph` — a named slice of a `HoloGraph`
- `flatten` — merges nested subgraphs into a single flat graph

### `fusion`

Optimization passes that rewrite the graph before scheduling:

- `constant` — folds compile-time constant nodes
- `cse` — common subexpression elimination
- `view_fusion` — merges adjacent `ElementWiseView` operations into a single pass

### `schedule`

Converts a fused graph into an execution schedule:

- `toposort` — Kahn's algorithm topological sort
- `levels` — assigns nodes to parallel execution levels (nodes at the same level have no data dependency)
- `critical_path` — identifies the longest dependency chain for latency analysis

### `builder`

Fluent builder for constructing `HoloGraph` programmatically without touching internal node/edge IDs.

### `constant`

`ConstantNode` — a graph node that holds a precomputed value, resolved at compile time.

## Example

```rust
use hologram_graph::builder::GraphBuilder;

let graph = GraphBuilder::new()
    .input("x", Shape::scalar())
    .op(PrimOp::Sigmoid, &["x"], "y")
    .output("y")
    .build()?;

let schedule = graph.schedule()?;
```
