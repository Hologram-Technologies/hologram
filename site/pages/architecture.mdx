# Architecture

## Dependency Graph

```
uor-foundation (git v3.5.0, traits only, no_std)
       │
   hologram-core       (LUT, views, ring, encoding — no_std + alloc)
       │
   hologram-graph      (expression graph, subgraphs, fusion, scheduling)
       │
   hologram-archive    (.holo format, rkyv zero-copy, mmap, entrypoints)
       │
   hologram-exec       (KV executor, buffer arena, parallel levels)
       │
   hologram-bench      (Criterion benchmarks)
```

Horizontal additions (no extra deps on core):

- **hologram-compiler** depends on hologram-graph
- **hologram-async** depends on hologram-exec + Tokio
- **hologram-ffi** depends on hologram-compiler + hologram-exec

## Design Principles

### O(1) Compute

All activation functions and pointwise ops are precomputed into LUT arrays at compile time. At inference time, each op is a single array index.

### Ring Algebra

Weights and activations are encoded as elements of integer rings (Q0, Q1). Arithmetic is closed within the ring — no float promotion needed.

### KV-Dispatch

Execution is a key-value dispatch: operation ID + operand encoding → precomputed result. The `hologram-exec` KV registry stores all reachable `(op, input) → output` pairs.

### LUT-GEMM

Matrix multiplication is decomposed into row-wise LUT lookups summed via an accumulator (`PsumBook`). Eliminates FLOP entirely for quantized weights.

### Zero-Copy Loading

Model weights are stored in `.holo` archives using `rkyv` for zero-copy deserialization. The loader maps the archive into memory and validates the header without copying bytes.

## Feature Gates

| Feature | Controls |
|---------|----------|
| `simd` | SIMD acceleration behind `#[cfg(target_arch)]` guards |
| `parallel` | Rayon parallel level execution in `hologram-exec` |
| `wasm` | Strips non-WASM deps; `hologram-core` compiles for `wasm32-unknown-unknown` |

## Conventions

- Max 3 function arguments; builder pattern for more.
- Functions ≤ 15 lines.
- `macro_rules!` for repeated trait implementations.
- No TODOs, stubs, or `unimplemented!()`.
- Single archive format version — no backwards compatibility shims.
