Hologram is a v2 rewrite targeting extreme inference speed by replacing floating-point arithmetic with integer LUT lookups and key-value dispatch, built on [`uor-foundation`](https://github.com/UOR-Foundation/uor-foundation) v3.5.0.

## Core Idea

Instead of computing `f(x)` at runtime, every activation and arithmetic operation is resolved as a table lookup:

```
result = LUT[encoded_input]   // O(1), integer index
```

Weights are quantized to integer ring elements. Matrix multiplication becomes a sum of LUT lookups (`LUT-GEMM`), eliminating floating-point entirely on hot paths.

## Capabilities

| Feature | What it does |
|---------|-------------|
| **Activation Lookup** | All activation functions precomputed into static arrays — each inference call is one integer index |
| **LUT-GEMM** | Matrix multiplication via LUT row-sum accumulation — no floating-point on hot paths |
| **Zero-Copy Loading** | `.holo` archives are memory-mapped and deserialized without copying bytes |
| **Graph Fusion** | Constant folding, CSE, and view fusion reduce ops before execution |
| **Parallel Execution** | Independent graph nodes execute in parallel across Rayon threads |
| **Async & Streaming** | Tokio-based async inference with incremental output streaming |
| **Cross-Platform** | `no_std` core, C ABI, and WASM bindings for embedded and browser targets |
