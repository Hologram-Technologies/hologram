All persistent types derive `rkyv::{Archive, Serialize, Deserialize}`. Single format version ‚Äî no backwards compatibility.

## Modules

### `format`

- `HoloHeader` ‚Äî file header with magic, version, and byte offsets for each section
- `HoloGraph` (archive representation) ‚Äî rkyv-serialized graph

### `loader`

- `MmapLoader` ‚Äî maps a `.holo` file into virtual memory; zero-copy access to weights
- `BytesLoader` ‚Äî loads from an in-memory `&[u8]` slice (useful for WASM)
- `LoadPipeline` ‚Äî chains header validation ‚Üí graph deserialization ‚Üí weight loading
- `LoadPlan` ‚Äî describes which sections to load (lazy loading support)

### `writer`

- `HoloWriter` ‚Äî serializes a `HoloGraph` + weights to a `.holo` byte stream
- `PipelineWriter` ‚Äî streaming writer for large models

### `section`

- `SectionTable` ‚Äî index of byte ranges for each section

### `weight`

- `WeightTensor` ‚Äî rkyv-serialized quantized weight array
- `quantize` ‚Äî converts `f32` weights to ring-encoded `u8` arrays

### `checksum`

SHA-256 integrity check on the weight section, verified at load time.

### `entrypoint`

- `ExecutionEntrypoint` ‚Äî bundles a loaded graph + schedule ready for `hologram-exec`

## Usage

```rust
use hologram_archive::loader::MmapLoader;

let loader = MmapLoader::open("model.holo")?;
let entrypoint = loader.load_entrypoint()?;
```
